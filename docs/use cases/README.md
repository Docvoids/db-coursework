
# Модель прецедентів

В цьому файлі необхідно перелічити всі документи, розроблені в проекті та дати посилання на них.

*Модель прецедентів повинна містити загальні оглядові діаграми та специфікації прецедентів.*



Вбудовування зображень діаграм здійснюється з використанням сервісу [plantuml.com](https://plantuml.com/). 

В markdown-файлі використовується опис діаграми

```md

<center style="
    border-radius:4px;
    border: 1px solid #cfd7e6;
    box-shadow: 0 1px 3px 0 rgba(89,105,129,.05), 0 1px 1px 0 rgba(0,0,0,.025);
    padding: 1em;"
>

@startuml

    right header
        <font size=24 color=black>Package: <b>UCD_3.0
    end header

    title
        <font size=18 color=black>UC_8. Редагувати конфігурацію порталу
        <font size=16 color=black>Діаграма прецедентів
    end title


    actor "Користувач" as User #eeeeaa
    
    package UCD_1{
        usecase "<b>UC_1</b>\nПереглянути список \nзвітів" as UC_1 #aaeeaa
    }
    
    usecase "<b>UC_1.1</b>\nЗастосувати фільтр" as UC_1.1
    usecase "<b>UC_1.2</b>\nПереглянути метадані \nзвіту" as UC_1.2  
    usecase "<b>UC_1.2.1</b>\nДати оцінку звіту" as UC_1.2.1  
    usecase "<b>UC_1.2.2</b>\nПереглянути інформацію \nпро авторів звіту" as UC_1.2.2
    
    package UCD_1 {
        usecase "<b>UC_4</b>\nВикликати звіт" as UC_4 #aaeeaa
    }
    
    usecase "<b>UC_1.1.1</b>\n Використати \nпошукові теги" as UC_1.1.1  
    usecase "<b>UC_1.1.2</b>\n Використати \nрядок пошуку" as UC_1.1.2
    usecase "<b>UC_1.1.3</b>\n Використати \nавторів" as UC_1.1.3  
    
    
    
    User -> UC_1
    UC_1.1 .u.> UC_1 :extends
    UC_1.2 .u.> UC_1 :extends
    UC_4 .d.> UC_1.2 :extends
    UC_1.2 .> UC_1.2 :extends
    UC_1.2.1 .u.> UC_1.2 :extends
    UC_1.2.2 .u.> UC_1.2 :extends
    UC_1 ..> UC_1.2.2 :extends
    
    
    UC_1.1.1 -u-|> UC_1.1
    UC_1.1.2 -u-|> UC_1.1
    UC_1.1.3 -u-|> UC_1.1
    
    right footer
        Аналітичний портал. Модель прецедентів.
        НТУУ КПІ ім.І.Сікорського
        Киів-2020
    end footer

@enduml

**Діаграма прецедентів**

</center>
```

| ID                 | <span id = CreateTask>`CreateTask`</span> |
| :----------------- | :--------------------------------- |
| Назва:             | Створити завдання (Create Task) |
| Учасники:          | Керівник команди, система |
| Передумови:        | 	- Керівник команди є адміністратором або власником проєкту <br>- Керівник команди авторизований |
| Результат:         | Створене завдання |
| Виключні ситуації: | - <font color="red">InvalidDataException</font>: Некоректно введені дані для завдання |
| Основний сценарій: | 1. Керівник команди переходить до сторінки проєкту <br>2. Керівник команди натискає "Створити завдання" <br>3. Керівник команди вводить дані для завдання  <br>4. Система перевіряє коректність даних (може виникнути <font color="red">InvalidDataException</font>) <br>5. Система створює нове завдання в проєкті |

@startuml
|Користувач|
    start;
    : Натискає кнопку "Створити завдання";

    |Система|
    : Виводить форму для створення завдання;

    |Користувач|
    : Вводить дані для завдання;
    
    |Система|
    : Перевіряє наявність обов'язкових полів;
    
    note right #lightcoral
    * Може виникнути
    <b>InvalidDataException</b>
    end note
    
    : Створює нове завдання в проєкті;
    : Повідомляє інформацію про завдання користувачам,\n яким воно було призначене;

     |Користувач|
        stop;
@enduml

| :----------------- | :--------------------------------- |
| Назва:             | Редагувати завдання (Edit Task) |
| Учасники:          | Користувач(керівник команди, учасник проєкту), система |
| Передумови:        | - Користувач має доступ до проєкту та до завдання <br>- Завдання існує в системі |

| Виключні ситуації: | - <font color="red">InvalidDataException</font>: Користувач ввів некоректні дані <br>- <font color="red">CancelException</font>: Користувач скасував підтвердження змін |
| Основний сценарій: | 1. Користувач обирає завдання і натискає кнопку "Редагувати завдання" <br>2. Користувач змінює дані (може виникнути <font color="red">InvalidDataException</font>) <br>3. Користувач натискає кнопку "Зберегти зміни" <br>4. Система запитує підтвердження змін (може виникнути <font color="red">CancelException</font>) <br>5. Система оновлює завдання та повідомляє користувача про успішне редагування |

    : Редагує завдання;

    |Система|
    : Перевіряє наявність обов'язкових полів;
    
    note right #lightcoral
    * Може виникнути
    <b>InvalidDataException</b>
    end note
    
    : Запитує підтвердження дії;
    
    |Користувач|
    : Натискає кнопку "Так";
    
    note left #lightcoral
    * Може виникнути
    <b>CancelException</b>
    end note    

    |Система|
    : Зберігає внесені користувачем зміни та \n відображає відредаговане завдання;
    : Повідомляє інформацію про завдання користувачам, \n яким воно було призначене;

    |Користувач|
        stop;

Система перевіряє наявність завдання (може виникнути <font color="red">TaskDeletedException</font>) <br>4. Система додає коментар до завдання та повідомляє користувача про успішне додавання |

    : Натискає кнопку "Додати коментар";
    
    note left #lightcoral
    * Може виникнути
    <b>TaskNoExistException</b>
    end note

    |Система|
    : Зберігає коментар;
    : Повідомляє користувача про успішне додавання;
